const vs let

const is used when the variable will not be assigned to anything else.

let is used when the variable may be reassigned later.

Why would I use const? I see it everywhere, but what if I decide later to change the variable? It seems let is much more flexible. What are the advantages of const over let?

Trying to reassign const will produce an error, which could be useful in debugging. This chapter says to use const to declare most variables.

Functions

You can send arguments through a function like an array.

function arguments() { return arguments; }

Call the function and enter the arguments (1,2,3,4,5) like parameters.

However, these can't be treated like arrays. Do make it an array, at ... right before the last arguement (no space).

Use arguments in a mean function.

function mean(...values) {
let total = 0;
for(const value of values) {
total =+ value;
}
return total/values.length;
}

Arrow Functions

const functionName = () => {function body};

for one-line functions, the {} aren't needed.
for one parameter, the () is not needed.
a return statement isn't required if it's the only statement.

The advantage to these is less code. Use these for short functions.

forEach()

Basically takes out the loop. I've seen them, and they make sense when I see them used, but when I have to use them... my mind turns to mush.

let array = [red, blue, yellow];
for (i = 0; i < array.length; i++) {
document.getElementById("output").innerHTML = `color ${i} is ${array[i]}`
}

Can instead be coded like this

array.forEach( (color,index) =>
document.getElementById("output").innerHTML = `color ${index} is ${color}`);

Note the arrow function didn't require {} because it was only one line.
