A loop simplified. I like how this is laid out.

  • 1. the for keyword
  • 2. a variable to keep track of the location you are at in the array
  • 3. a check to see if the loop should continue
  • 4. an update that increases the location tracking variable by one
  • 5. the lines of code to execute every time you go around the loop
for(i = 0; i < x; i++){ ... }

+= appends whatever is on the left. 17 =+ 3 will make 20. So displaying a result from javascript like this

document.getElementById("display").innerHTML += result;

will take whatever result is and add it to whatever is already in "display". Just using = will assign it, not append it.

Mobile UX

On small mobile devices, keep things easy to reach (reachability), bottom right of the surface. Tablets, keep things on the bottom half. For example, put the nav bar on the bottom.

Avoid using the hamburger menu icon. Add or use the word "menu" to improve user experience.

How to speed things up

Reuce image usage throughout sites and apps. SVG webfonts are lower bandwidth Optimize and minify css and js files GZIP fields to eliminate unnecessary data.

Mobile Forms

Use float labels inside text input fields to save space. These are like placeholder text, but they are small and they don't disappear when the user inputs information in the field.
