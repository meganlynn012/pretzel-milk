Debugging

Coding in strict mode is a good idea. This can help catch errors early on that may impact code later. It also encourages better quality of coding.

To activate strict mode, add the following line to a js file.

'use strict';

This line can even be used inside a function so it only applies to code inside that block, which is recommended over applying strict mode to the entire js file. This helps when working with someone else's code.

When using APIs, and I think anything that would require specific browser features, it is a good idea to have the browser check for those features before it does something that relies on those features being there.

Is it a good idea to do an if statement to check that the screen isn't too small? I've seen that done before. Or would be better to do that in CSS?

So this is where the alert() comes in handy. Use alert() for debugging a code. The code will stop running when an alert() is activated. It won't begin again until the user clicks 'OK'. This helps check that variables and if statements are running properly.

function amIOldEnough(age){
if (age = 12) {
alert(age);
return 'No, sorry.';
} else if (age < 18) {
return 'Only if you are accompanied by an adult.';
}
else {
return 'Yep, come on in!';
}
}

There is a bug in that code. The alerts will help find it. age = 12 should be age < 12. Testing with alerts will help find this bug.

Use Try, Catch, & Finally

Running code in a try block will make it run normal, but if there is an exception (error) the catch part of the code will activate. Finally will run no matter what the outcome of the try and catch.
